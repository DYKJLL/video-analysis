## YouTube
- **方案**：yt-dlp `--write-auto-sub --write-sub --convert-subs=srt --skip-download`
- **代理**：`http://172.23.32.1:7897`（Windows Clash Verge，WSL 可达 ✅）
- **结果**：✅ 成功，字幕文件生成 `/tmp/yt_sub.{lang}.vtt`，Rick Astley 字幕 4017字
- **注意**：youtube-transcript-api 在本环境 IP 被墙，不可用；yt-dlp 可用
- **代理警告**：yt-dlp 需在 Python 代码内设置 `os.environ['https_proxy']`，不从 shell 环境继承

## Bilibili
- **方案**：yt-dlp `--list-subs` / `--write-sub --convert-subs=srt --skip-download`
- **代理**：`http://172.23.32.1:7897`
- **结果**：⚠️ 可连接，但大多数视频无字幕（弹幕 danmaku ≠ 字幕 subtitle）
- **高清限制**：1080P 高码率需要登录会员；免费可获取 720P 及以下
- **字幕格式**：官方字幕可用；用户投稿字幕需登录

## 抖音
- **方案**：抖音公开 API `https://www.douyin.com/aweme/v1/web/aweme/detail/`
- **结果**：❌ HTTP 200 但响应 body 为空（被反爬拦截）
- **yt-dlp 方式**：需要 Fresh cookies（`ERROR: Fresh cookies required`）
- **原因**：Chrome Cookie 在 WSL 无法读取（DPAPI 加密隔离，WSL Chrome DB 为空）
- **✅ 新方案（2026-05-15 验证成功）**：Playwright 启动独立 Chromium
  - WSL 无法直接连接 Windows Chrome 调试端口（`172.23.32.1:9222` TCP可达但HTTP被防火墙拦）
  - Playwright 启动自己的 Chromium，走 `http://172.23.32.1:7897`（Windows Clash Verge 代理）
  - 可获取：标题/描述/章节时间戳/评论区/统计数据
  - **不需要任何登录 Cookie**，完全绕过反爬
- **实测**：`v.douyin.com/iefjhG8Z_po/` → 成功获取完整内容
  - 标题：gstack + Superpowers：cc绝配编程提效
  - 章节时间戳：8个章节（引言/gstack/Superpowers/互补性/具体用法/任务分流/CLAUDE.md/个人心得）
  - 评论区：包含作者回复
  - 统计数据：7966赞/360评论/1.0万观看

## X / Twitter
- **方案**：yt-dlp 直接支持（`https://x.com/user/status/id` 或 `https://twitter.com/user/status/id`）
- **代理**：`http://172.23.32.1:7897`
- **结果**：✅ 代码正确，实测可提取个人资料页内容
- **评论抓取**：需要 Apify actor `apify~twitter-chrome-scraper`（需 Token）
- **直接抓取**：X 移动端 API `https://api.twitter.com/2/tweets/{id}` 有登录墙

## 代理说明（v1.5.1 重要更新）

- WSL 直连外网不通（DNS/路由问题）
- **正确出口**：`http://172.23.32.1:7897`（Windows Clash Verge）——WSL 可达，GitHub 返回 200
- 代码中必须显式设置 `os.environ['http_proxy']` 和 `os.environ['https_proxy']`
- `youtube-transcript-api` 内部使用 `requests`，不继承 shell 环境变量，必须代码内设置
- **注意**：mihomo in WSL（`/root/.local/share/hermes-proxy/`）的 VLESS Reality 节点与 curl/requests TLS 不兼容（SSL EOF 错误），用 Windows Clash Verge 代理更可靠
